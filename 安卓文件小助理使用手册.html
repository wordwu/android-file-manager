安卓文件小助理 — 使用手册

1. 软件介绍

安卓文件小助理是一款 macOS 上的 Android 设备文件管理工具，通过 ADB（Android Debug Bridge）连接你的手机，实现文件浏览、复制粘贴、删除重命名、应用管理、APK 安装等功能。

无需 root，无需在手机上装任何 App，只需要一根数据线（或无线调试），就能像操作 Mac 本地文件一样操作手机文件。

运行环境：macOS 15.0（Sequoia）及以上，Apple Silicon（M 系列芯片）

2. 软件用法

2.1 主界面概览

打开软件后，左侧是设备列表栏，右侧是文件浏览区。顶部有两排工具栏：

第一排工具栏：后退、向上、主页、刷新、复制、剪切、粘贴、删除、下载到 Mac、安装 APK、传输任务、侧边栏

第二排工具栏：搜索框（实时搜索，输入即触发）、应用管理按钮、列表/图标视图切换、排序方式、文件类型过滤、批量重命名

2.2 文件浏览

  • 双击文件夹进入，双击文件预览
  • 单击选中单个文件，Cmd+点击多选
  • 右键菜单提供复制、删除、下载等快捷操作
  • 拖拽 Mac 文件到窗口可上传到当前手机目录

2.3 文件操作

操作说明
复制 (Cmd+C)将选中文件放入剪贴板
剪切 (Cmd+X)将选中文件放入剪贴板（移动到新位置）
粘贴 (Cmd+V)将剪贴板文件粘贴到当前目录
删除 (Delete)删除选中的文件或文件夹
重命名点击"重命名"按钮，支持四种批量模式
下载将选中文件从手机复制到 Mac 桌面
拖拽上传将 Mac 文件拖到文件列表即可上传

批量重命名四种模式：

  1. 序号重命名：将文件按顺序重命名为 前缀_001、前缀_002……，支持自定义前缀和起始编号
  2. 查找替换：将文件名中的指定文字替换为其他文字
  3. 添加前后缀：在文件名前或文件名后（扩展名前）添加文字
  4. 正则替换：用正则表达式匹配并替换文件名

2.4 搜索文件

在搜索框输入关键词，自动在当前目录和所有子文件夹中递归搜索。搜索结果按文件夹优先排列。按 Esc 或点击搜索框右侧的叉号清除搜索。

2.5 排序与过滤

  • 排序：按名称、大小、修改时间、类型排序
  • 过滤：显示全部 / 仅文件夹 / 图片 / 视频 / 音频 / 文档 / APK 安装包

2.6 应用管理

点击工具栏的「应用管理」按钮（九宫格图标）：

  • 查看手机上所有已安装的应用
  • 按名称搜索应用
  • 切换显示/隐藏系统应用
  • 右键应用可卸载、查看详情、提取 APK

2.7 安装 APK

两种方式：

  1. 在文件管理中找到 APK 文件，选中后点击「安装 APK」按钮
  2. 将 Mac 上的 APK 文件拖拽到应用管理窗口

安装过程会显示进度和结果。如果安装失败，会显示具体原因（如版本冲突、空间不足、签名不一致等）。

2.8 传输任务

点击第一排工具栏最右侧的「传输任务」按钮，查看当前正在进行的文件传输进度，包括上传到手机和下载到 Mac 的任务。支持取消进行中的任务。

2.9 书签

右键文件列表中的文件夹 →「添加书签」，之后在路径栏旁边可以快速跳转到已收藏的目录。右键书签可删除。

2.10 屏幕镜像

点击工具栏的「屏幕镜像」按钮，会弹出独立的手机屏幕镜像窗口：

  • 实时画面：显示手机屏幕内容，延迟约 40-60ms
  • 鼠标操控：鼠标点击 = 手指触控，鼠标滑动 = 手势操作
  • 键盘输入：先点一下手机画面中的输入框，再用电脑键盘打字
  • 右键菜单：弹出 Home / Back / Recent Apps 快捷操作

键盘输入前提：手机需开启「开发者选项 → USB 调试（安全设置）」。小米/HyperOS 用户在开发者选项里找到这项开关打开，拔插 USB 重新连接即可。

3. 打开时可能出现的问题

3.1 「安卓文件小助理已损坏，无法打开」

原因：macOS Gatekeeper 阻止了未签名的应用。

解决：

  1. 打开「系统设置」→「隐私与安全性」
  2. 往下滚动，找到「已阻止使用 XXX，因为来自身份不明的开发者」
  3. 点击「仍要打开」
  4. 如果上述步骤不行，打开「终端」，执行：
xattr -cr /Applications/安卓文件小助理.app

或（如果放在桌面）：

xattr -cr ~/Desktop/安卓文件小助理/安卓文件小助理.app

3.2 提示「未找到 adb」

原因：adb 工具未能正确内嵌到 App 中。

解决：重新下载完整版的 App，确保 App 体积在 14MB 左右（内含 adb 二进制约 19MB）。不要使用轻量版或通过其他渠道下载。

3.3 软件打开了但看不到设备

原因：手机上的 USB 调试没有开启，或者未授权该电脑。

解决：

  1. 确认手机已通过数据线连接到 Mac
  2. 手机上应该弹出「允许 USB 调试吗？」对话框，勾选「始终允许」并点击确定
  3. 如果没弹出，关闭 USB 调试再重新打开，然后重新插拔数据线
  4. 检查数据线是否支持数据传输（不是仅充电线）

3.4 能连接但提示「目录无法读取 / Permission denied」

原因：Android 11+ 的 scoped storage 限制，部分系统目录无法通过普通 ADB 访问。

解决：使用 /sdcard/ 下的目录（Download、Documents、DCIM 等），这是普通用户可以完全读写的区域。/storage/emulated/0/ 等同于 /sdcard/。

3.5 无线调试无法连接

确认以下条件：

  • 手机和 Mac 连接的是同一个 WiFi（同一个路由器）
  • 手机已先通过 USB 线开启一次无线调试（点击侧边栏的 WiFi 图标）
  • 手机 Android 版本 ≥ 11

如果仍然连接不上：

  • 在手机上打开「开发者选项」→「无线调试」，看端口号是多少
  • 手动在 Mac 终端执行：adb connect 手机IP:端口号
  • 如果手动能连上但 App 连不上，可能是防火墙/路由器隔离了设备间通信

3.6 OPPO / 一加 / realme 设备连接后不显示型号

原因：这些品牌的固件对型号信息的解析有特殊规则。

解决：无需处理，不影响功能使用。App 会自动尝试多种方式获取设备名称，显示为「Unknown Device」时不影响文件操作。

4. 各品牌手机 USB 调试设置指南

4.1 小米 / Redmi（MIUI / HyperOS）

开启开发者选项：

  1. 进入「设置」→「我的设备」→「全部参数与信息」
  2. 连续点击「MIUI 版本」（或 HyperOS 版本）7 次
  3. 提示「您已处于开发者模式」

开启 USB 调试：

  1. 「设置」→「更多设置」→「开发者选项」
  2. 打开「USB 调试」
  3. 打开「USB 安装」（允许通过 USB 安装应用）
  4. 关键：打开「USB 调试（安全设置）」— 部分小米机型必须开启此项才能正常工作
  5. 关闭「MIUI 优化」（可选，若遇到问题建议关闭）

注意事项：MIUI/HyperOS 的 USB 调试有 10 分钟超时限制，超时需重新授权。

4.2 华为 / 荣耀（HarmonyOS / EMUI / MagicOS）

开启开发者选项：

  1. 「设置」→「关于手机」
  2. 连续点击「版本号」7 次

开启 USB 调试：

  1. 「设置」→「系统和更新」→「开发人员选项」
  2. 打开「USB 调试」
  3. 打开「仅充电模式下允许 ADB 调试」

注意事项：华为默认 USB 连接模式为「仅充电」，务必打开上方的第 3 项，否则无法连接。连接后手机弹出 RSA 密钥指纹对话框，勾选「始终允许」并确认。

4.3 OPPO / 一加 / realme（ColorOS / OxygenOS / realmeUI）

开启开发者选项：

  1. 「设置」→「关于本机」→「版本信息」
  2. 连续点击「版本号」7 次
  3. 输入锁屏密码验证

开启 USB 调试：

  1. 「设置」→「其他设置」→「开发者选项」
  2. 打开「USB 调试」

注意事项：OPPO 设备连接后会弹出 10 位数字验证码，必须在手机上点击确认。不要关闭这个弹窗，否则无法连接。

4.4 vivo / iQOO（OriginOS / Funtouch OS）

开启开发者选项：

  1. 「设置」→「系统管理」→「关于手机」→「版本信息」
  2. 连续点击「软件版本号」7 次

开启 USB 调试：

  1. 「设置」→「系统管理」→「开发者选项」
  2. 打开「USB 调试」

注意事项：vivo 连接后手机顶部会弹出「USB 调试授权」通知，需要下拉通知栏点击确认。

4.5 三星（One UI）

开启开发者选项：

  1. 「设置」→「关于手机」→「软件信息」
  2. 连续点击「版本号」7 次

开启 USB 调试：

  1. 「设置」→「开发者选项」
  2. 打开「USB 调试」
  3. 弹出 RSA 密钥对话框时，勾选「始终允许使用这台计算机进行调试」

4.6 Google Pixel / 原生 Android

开启开发者选项：

  1. 「设置」→「关于手机」
  2. 连续点击「版本号」7 次

开启 USB 调试：

  1. 「设置」→「系统」→「开发者选项」
  2. 打开「USB 调试」

Pixel 系列通常是最省心的，插上即可连接。

4.7 其他品牌（摩托罗拉、中兴、努比亚、联想等）

步骤大同小异：在「关于手机」里连点 7 次版本号 → 开发者选项里开启 USB 调试。

通用检查清单：

  • 开发者选项是否处于开启状态（顶部开关是蓝色/绿色）
  • USB 调试是否勾选
  • 数据线是否支持数据传输（换一根线试试）
  • 连接后手机屏幕是否有弹窗、通知要求确认

5. 无线调试设置

无线调试可以让你摆脱数据线，通过 WiFi 连接手机。

前提条件：

  • 手机 Android 11 及以上
  • 手机和 Mac 连在同一个 WiFi 网络
  • 第一次使用需要先用 USB 线连接一次

操作步骤：

  1. 先用 USB 连接手机，确保 USB 调试已授权
  2. 在软件左侧设备列表中，点击设备旁边的 WiFi 图标
  3. 等待几秒，手机会自动切换到无线模式
  4. 拔掉 USB 线，设备应仍然在线
  5. 如果显示已断开，点击设备右键 →「连接无线设备」手动输入 IP 和端口

注意事项：

  • 部分路由器开启了 AP 隔离（设备间不能互相通信），无线调试会失败
  • 手机息屏后 WiFi 可能断开，导致 ADB 掉线
  • 建议在「开发者选项」中开启「充电时保持唤醒」

8. 常见问题速查

Q: 应用提示「设备离线」但手机明明连着？

A: 检查手机上的 USB 调试授权是否过期。拔掉数据线，关闭 USB 调试再重新打开，重新连接，注意看手机屏幕上的授权弹窗。

Q: 能连上但不能读写某些目录？

A: Android 11+ 的权限限制。请使用 /sdcard/ 下的目录，不要尝试访问 /data/ 等系统目录。

Q: 传输大文件很慢？

A: USB 2.0 理论速度约 30MB/s，WiFi 更慢。大文件建议使用 USB 有线连接传输。具体速度取决于数据线和手机 USB 接口版本。

Q: 无线调试总是掉线？

A: 检查 WiFi 是否开启了 AP 隔离（客户端隔离），或者手机是否在 WiFi 信号较弱的位置。部分路由器在设备空闲时会断开连接。

最后：如果以上方法都无法解决你的问题，请截图错误信息，连同你的手机品牌和型号发给开发者。不要用「不能用了」「不好使」这种反馈——有图有真相，才能秒定位。
