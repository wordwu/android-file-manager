# 受够了 Android File Transfer？这个免费工具把安卓手机的「文件」做成了 Mac 版 Finder

Android 用户在 Mac 上传输文件有多痛苦，经历过的人都懂。

Google 官方的 Android File Transfer——那个 2012 年最后一次更新的黄色图标软件——至今还在用。卡顿、断连、文件名乱码、文件夹一多就崩溃。试过 OpenMTP 的人会发现界面过于复古。其他的要么收费、要么要装手机端 App、要么只支持 WiFi 传输速度感人。

而我只想要一件事：**插上线缆，像 Finder 一样操作手机里的文件。**

所以我用 SwiftUI 自己写了一个。开源、免费、不需要手机上装任何 App。叫「安卓文件小助理」。

---

## 插上去，就是一个 Finder

连上手机（USB 或同 WiFi 下的无线 ADB），左侧出现你的设备。点击进入文件系统——文件夹树、文件列表、右键菜单，所有操作逻辑和 Finder 一致。

比较意外的是 **adb pull / push 的传输速度比 MTP 快得多**——MTP 的设计决定了它在大批量小文件传输时性能很差，而 ADB 不存在这个问题。一个 2GB 的文件夹，MTP 可能要 15 分钟，ADB 方式不到 2 分钟。

支持的日常操作包括：

- **复制 / 剪切 / 粘贴**：完全照搬 Finder 的快捷键逻辑（⌘C / ⌘X / ⌘V）
- **拖拽上传**：从 Mac 桌面直接拖文件到 App，自动上传到当前手机目录
- **Cmd+D 下载**：选中文件或文件夹，一键下载到 ~/Downloads
- **同名文件自动编号**：重复文件下载不会覆盖，自动加后缀（像 macOS 自己的逻辑一样）
- **图片预览**：点击手机上的图片文件，右侧弹出预览面板

## 搜索和批量重命名

搜索功能设计得比较认真：**输入即搜索，300ms 防抖**。不需要手动按回车或点搜索按钮，打字就会自动在当前目录及所有子文件夹中递归匹配。结果按名称、修改日期、大小排列。

批量重命名提供了四种模式：序号（IMG_001、IMG_002…）、查找替换（把「副本」统一改成日期）、前缀后缀批量添加、以及正则表达式（给懂的人准备的）。

这个功能的细节打磨了很久——因为手机上的文件命名往往没有规律，四种模式基本覆盖了我能想到的所有场景。

---

## APK 管理：安装 + 提取 + 批量备份

应用管理面板读取已安装的三方应用列表，支持两个操作：

1. **安装**：从 Mac 上选择一个 .apk 文件，直接安装到手机（跳过微信文件大小限制、跳过 QQ 传输过期问题）
2. **提取**：把手机上已安装的 app 导出为 APK 到 Mac

还有一个最近加的批量备份功能——点击「备份应用」，弹窗显示所有三方应用，默认全选，点确认后自动创建 `~/Downloads/<手机型号>_<日期>/` 文件夹，把所有 APK 复制到 Mac。备份过程中有进度条和当前文件名提示。

这个功能来自一个很真实的场景：换手机的时候想把老手机上的 app 都备份下来。以前没有这个功能时只能在手机上一个个导出，现在一键搞定。

---

## 设备信息面板

左上角有个独立的信息面板，显示三类数据：

- **电池**：电量百分比 + 充电状态 + 温度 + 健康度
- **存储**：内部存储和 SD 卡（如果有）的已用/可用/总量，带环形图
- **系统**：品牌、型号、Android 版本、SDK 级别、内核版本

给需要排查问题、或者帮家人远程看手机状态的场景准备的。

## 屏幕镜像：安卓版「iPhone 镜像」

这是 v4.0 的重头戏。底层用的是 scrcpy——Android 生态最成熟的屏幕镜像工具。我把它内嵌到了 App 里，点一下工具栏的「屏幕镜像」，弹出一个独立窗口显示手机实时画面。

和 iPhone 镜像对比一下：

| | iPhone 镜像 | 安卓文件小助理 |
|---|---|---|
| 支持跨品牌 | 仅 Apple | 小米/华为/OPPO/vivo/三星…全支持 |
| 鼠标操控手机 | ✅ | ✅ |
| 键盘输入 | ✅ | ✅（需开「USB调试安全设置」） |
| 录屏 | ❌ | ✅ |
| 剪贴板同步 | 单向 | 双向 |
| 音频转发 | 手机上播放 | ❌（Android 11+ 限制） |
| 延迟 | 极低 | 40-60ms |
| 价格 | 随系统 | 免费 |

实际使用中延迟基本不可感知。鼠标点画面里的按钮、滑动列表、拖拽文件——跟在手机上操作几乎没区别。快捷键也完整：Option+H 回主页、Option+B 返回、F11 全屏。

有一点要注意：小米手机需要额外在开发者选项里打开「USB 调试（安全设置）」，否则电脑键盘打不出字（鼠标不受影响）。其他品牌一般不需要额外步骤。

## 技术花絮

项目用 SwiftUI + MVVM 架构，37 个源文件，约 6500 行 Swift。ADB 解析的边界情况打磨了比较久——不同品牌手机的 `ls` 输出格式差异很大，小米 15 用的是 toybox 的 ISO 日期格式（`2026-06-16 14:30`），三星和 OPPO 各自有变种。最终用了四种解析策略做兼容判断。

测试方面：4 个测试文件、34 个用例全部通过（`swift test`）。主要覆盖 ADB 输出解析、文件策略回退、粘贴重命名冲突处理。新加的备份和屏幕镜像功能还没补测试——这是接下来要填的坑。
